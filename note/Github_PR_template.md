# Pull-Request Template

## 프로젝트 적용 방법

.github/PULL_REQUEST_TEMPLATE.md 작성

## 양식

```
## 제목
[풀 리퀘스트에 간결하고 설명적인 제목을 입력하세요. 예: "API 호출에 대한 캐싱 메커니즘 개선" ]

## 변경 유형
- [ ] 새로운 기능
- [ ] 버그 수정
- [ ] 문서 업데이트
- [ ] 리팩토링
- [ ] 핫픽스
- [ ] 보안 패치
- [ ] UI/UX 개선

## 설명
[변경한 내용에 대한 자세한 설명을 입력하세요. 이러한 변경 사항의 이유와 관련 컨텍스트를 포함하세요. 관련 문제를 링크하세요.]
```
